Setting up your computer for development

Microsoft also has specific instructions for how to do it here.

You need to do a Windows computer to run this tutorial, as only Windows computers can run the Windows Application Projects that this project depends on.

Installing Visual Studio

The first step you need to do is installing Visual Studio for your computer. Keep in mind that you must do it in a Windows computer, as Apple computers do NOT have support for Windows Application Projects.

Click on this link and select the Community version for Windows computers.

Follow the steps for installing Visual Studio on your computer. When Visual Studio Installer asks about which modules to use in your program, make sure to select Windows Application Projects for C++ and C#

Installing Windows Application Project to Visual Studio (skip if you installed Visual Studio using the above steps)

Note that you need Visual Studio 2019 or higher to connect to the HoloLens. If you do not have that, please install a higher version.

Download the Visual Studio installer for the Community version for Windows computers.

Open the Visual Studio installer

Click on modify for the Visual Studio version that you want to install Windows Application projects on

Under the Desktop and Mobile category, click on Windows Windows Platform Development

Click on Modify to install the module into your Visual Studio version

Replacing the fo-dicom dll file

The HoloLens runs on ARM, which is different from the x86 architecture your computer probably runs. Therefore, you need to change the debug settings from x86 to ARM.

To do this, you need to first find the .nuget folder in C:/Users/[Windows username].

Navigate to .nuget/packages/fo-dicom.universal/4.0.x/runtimes

From here, go to win10-arm/lib/uap and copy the "Dicom.Core.dll" file

Paste the file into .nuget/packages/fo-dicom.universal/4.0.x/lib/uap and then select yes to replace the file already there.

Open Visual Studio and switch the deploy/debug settings from x86 to ARM (it should be on the menu right next to the green run button

In order to revert these changes, instead of copying the Dicom.Core.dll file in win10-arm, copy the Dicom.Core.dll file in win10-x86 into the library
